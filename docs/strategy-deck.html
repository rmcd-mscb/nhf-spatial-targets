I / VIII
The Mission

Turn gridded earth-system data into calibration targets for the National Hydrologic Model.

Five targets defined in USGS TM 6-B10 — runoff, actual ET, recharge, soil moisture, and snow-covered area — are built by spatially aggregating public gridded datasets onto an HRU fabric with gdptools. Each target is a normalized range of plausible values, not a single estimate.

Many sources, one fabric, one reproducible answer.

II / VIII
Architecture

Projects are fabric-specific.
The datastore is shared.

◆ DATASTORE — shared

Raw consolidated data, fabric-agnostic

One directory per source: merra2/, era5_land/, reitz2017/… Fetch writes here once; every project that points at the same datastore reuses the data.

◇ PROJECT — per fabric

config · manifest · weights · targets

Holds config.yml, fabric.json, credentials, the run manifest, aggregated outputs, weight caches, and final calibration targets — never deleted; it is the audit trail.

Two projects (GFv1.1 and GFv2.0, say) can share a single datastore on a large drive while their project directories live wherever the modeler wants them.

V / VIII
Fetch Routines

One module per source.
One shape.

  • Metadata (URL, product ID, bbox, variables) comes only from catalog.py.
  • Downloads land in <datastore>/<source_key>/ — never inside a project.
  • Credentials resolved from .credentials.yml → ~/.netrc, ~/.cdsapirc.
  • Idempotent: mtime checks + atomic rename — never re-download what’s already current.
  • Annual splits where APIs demand it (ERA5-Land: 12 monthly CDS requests per year).
  • Every module has a matching tests/test_<module>.py.
# fetch/era5_land.py — abridged def fetch(project, year): src = catalog.get_source("era5_land") dest = project.datastore / src.key for month in range(1, 13): chunk = _download_chunk(src, year, month) _atomic_write(dest / chunk.name, chunk) project.manifest.record(src, year)

VI / VIII
Consolidation

From raw chunks to one CF-1.6 NetCDF per source.

◆ CF-COMPLIANT

Standards in, standards out

Explicit CRS, grid_mapping, units, coordinates, cell methods — so downstream tools (gdptools, xarray) never guess.

◆ ATOMIC

Write-then-rename

Consolidated NCs are written to a temp path and renamed on success. A killed run never leaves a half-valid file behind.

◆ INCREMENTAL

mtime-aware re-aggregation

Daily NCs rebuild only when any hourly input is newer; the monthly roll-up follows. Re-runs are near-free when nothing has changed.

The consolidated NC is the hand-off point — aggregation (later) reads it and nothing else.
